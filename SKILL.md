# 单点场景生图 PPT

## 定位

这个 skill 用于制作**单点沟通场景**下的高视觉图片型 PPT。它通过对话先把目标、受众、风格和约束问清楚，再使用 imagegen 生成带文字的整页幻灯片图片，最后裁切、叠加真实 Logo，并封装成 PPTX、PDF 或每页 PNG。

这个 skill 不按行业、主题或内容类型触发，而按交付方式判断是否适合：**单点沟通、重视觉、接受图片化、不强调后续频繁编辑**。

## 适配判断

当以下条件基本成立时，使用这个流程：

- 这份材料围绕一个明确沟通目标展开，不是多主题资料合集。
- 用户重视整体观感、统一风格、缩略图效果和现场讲解体验。
- 用户接受正文和图表以图片形式存在，不要求所有元素可编辑。
- 材料用于正式汇报、售前交流、内部同步、方案介绍、培训讲解、复盘展示等场景。
- 用户希望通过多轮对话逐步确认风格、页数、详略、输出格式，并保留逐页提示词便于复用。

当用户主要需要以下能力时，不应把这个 skill 作为主路径：

- 财务报表、经营复盘、指标看板等强数据精确材料。
- 后续需要频繁改文字、改表格、改图表的材料。
- 必须严格遵循企业母版、字号、边距、版式规范的正式模板文件。
- 文字非常密集、脚注很多、长表格很多的留档型文档。
- Logo、品牌、法律披露有像素级要求，且不能通过后处理校正。

如果不适配，直接说明原因，并建议改用可编辑 PPT、文档或数据报告流程。

## 先澄清

生成图片前，先确认会显著影响交付结果的事项。用户已经说明清楚时，可以直接复述假设后推进。

必须确认或锁定：

- 使用场景：正式汇报 / 售前交流 / 内部同步 / 方案介绍 / 培训讲解 / 复盘展示 / 其他。
- 沟通目标：让对方理解 / 认同方案 / 支持决策 / 进入 POC / 形成共识 / 展示成果 / 交接留存。
- 受众类型：高层 / 业务方 / 技术方 / 运营方 / 客户 / 内部团队。
- 页数范围：短版 / 标准版 / 详版，并转成明确页数。
- 内容详略：轻量视觉版 / 标准汇报版 / 详版留存版。
- 视觉风格：白底商务 / 科技感 / 深色大屏 / 咨询报告 / 产品发布 / 轻营销 / 模板派生。
- 模板要求：沿用模板 / 只提取配色和 Logo / 不使用模板。
- Logo 策略：每页 Logo / 只封面尾页 / 不加 Logo。
- 输出格式：PPTX / PDF / 每页 PNG / 逐页提示词 / QA 记录 / 来源证据清单。
- 禁止内容：客户名、内部指标、真实样本、报价、敏感案例、未经确认来源、模型参数等。

## 标准流程

1. **适配与简报**
   - 判断是否适合图片型 PPT。
   - 把用户需求压缩成一句明确沟通目标。
   - 记录假设：受众、页数、风格、输出、敏感边界。

2. **资料与结构**
   - 读取用户提供的本地材料、模板、笔记或参考图。
   - 只有在用户需要当前公开事实或来源证据时才联网。
   - 先生成逐页大纲：每页标题、页面目标、2-4 条核心信息。
   - 大纲或风格不确定时，先让用户确认，再批量生图。

3. **风格系统**
   - 有模板时，提取配色、Logo、字体气质、版式节奏，但不要照搬模板瑕疵。
   - 没有模板时，根据使用场景选择或询问视觉方向。
   - 明确 Logo 后处理位置。不要让 imagegen 绘制真实 Logo。

4. **逐页提示词**
   - 每页都保留详细提示词。
   - 使用 `references/prompt-patterns.md` 的结构。
   - 每页必须包含：精确标题、核心文字、版式结构、视觉元素、风格提示词、Logo 规则、负向约束。

5. **生成整页图片**
   - 调用 `imagegen`，每页单独生成 16:9 带文字整页图。
   - 每张原始生图都保存到项目目录。
   - 风格风险较高时，先生成 1-3 页给用户或自己检查，再批量继续。
   - 出现文字不可读、结构跑偏、伪 Logo、明显风格不一致时，重抽该页。

6. **后处理与封装**
   - 将所有图片裁切或缩放到统一 16:9。
   - 遮盖任何可见的 Logo 预留框或占位文字。
   - 叠加真实 Logo 或用户提供的品牌素材。
   - 封装为图片型 PPTX，并按需要导出 PDF 或每页 PNG。
   - 环境合适时使用 `scripts/package_image_deck.py`。

7. **QA**
   - 生成 contact sheet 缩略图。
   - 检查页数、尺寸、Logo 位置、是否有伪 Logo、文字可读性和明显敏感信息泄露。
   - 能导出时，渲染 PPTX 或导出 PDF，确认页数和画面正常。
   - 最终说明残余风险，尤其是 imagegen 中文错字、模板一致性和 Logo 遮盖瑕疵。

## 默认交付物

默认输出：

- 图片型 `.pptx`
- 逐页详细提示词
- contact sheet 缩略图
- QA 检查记录

可选输出：

- PDF 版
- 每页最终 PNG
- 来源与证据清单
- 后续可编辑版重建建议

## 硬性规则

- 不要把图片型 PPT 说成可编辑 PPT，除非文字和图表确实是原生对象。
- 不要让 imagegen 生成真实 Logo、合作方标识、证书、二维码、合规印章或品牌符号。
- 最终页里不能保留“Logo 预留区”之类的占位文字；后处理时要先遮盖再叠加真实 Logo。
- 不要编造数字、引用、标准、客户名或证据。
- 面向公开复用时，示例必须通用化，不能包含私人客户材料或内部项目口径。
